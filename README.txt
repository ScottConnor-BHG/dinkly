-------------------------------------------------------------------------------------------------------------------------------
Dinkly CLI Tools
-------------------------------------------------------------------------------------------------------------------------------

	Generate Dinkly datamodel files (consumes all yml files under config/schema + will not overwrite existing custom classes):

		php tools/gen_models.php

	Generate a new Dinkly module:

		php tools/gen_module.php -m <module name>

	Load fixtures (preloads tables with data stored in yml files under config/fixtures)

		php tools/load_fixtures.php

-------------------------------------------------------------------------------------------------------------------------------
Architecture Overview
-------------------------------------------------------------------------------------------------------------------------------

classes/												= All your PHP classes go under this somewhere
	core/												= Core Dinkly classes
	models/												= Datamodel classes
		base/											= Autogenerated base datamodel classes (it's probably best you don't touch these)
		custom/											= Autogenerated custom datamodel classes (make your changes to these instead)
	thirdparty/										    = Put your thirdparty classes here
		Yaml/											= Needed for parsing the Yaml config files
		Postmark/										= A git submodule, linked to a Postmark PHP API (https://github.com/Znarkus/postmark-php.git)

config/													= Global configuration files
	fixtures/											= Data fixtures
	schema/												= Datamodel configuration files
	db.yml  											= Database configuration file

layout/													= Primary HTML layouts
	footer.php 											= The last HTML to appear on every page; can be disabled in the controller
	header.php    										= The first HTML to appear on every page; can be disabled in the controller

modules/												= Dinkly modules
	test/												= The api controller for the quiz application, Backbone chats with this via RESTful interaction
		views/											= Dinkly views, always defaults to, well, default.php
		TestController.php 								= The primary api controller, lots of good stuff in here

tools/													= CLI tools and basic tests

web/													= Web accessible directory
	css/												= Yep
	font/												= Font files, some of these are used by the Twitter Bootstrap
	images/												= Yep
	js/													= Ahem, yep
